# DECISION LOG\n\n## DEC-001: Universal Framework + Policy Profiles\n**Date:** 2026-04-22\n**Status:** Accepted\n**Context:** Should we build separate per-platform eval tools or one universal framework?\n**Decision:** Universal eval core with pluggable platform policy profiles (YAML).\n**Rationale:** Maximizes reusability. Same scoring engine works across any platform by swapping policy profiles. Portfolio piece has higher impact as a universal tool.\n\n## DEC-002: TikTok + RedNote as First Profiles\n**Date:** 2026-04-22\n**Status:** Accepted\n**Context:** Which platforms to target first?\n**Decision:** TikTok and RedNote (Xiaohongshu) as first two policy profiles.\n**Rationale:** Covers both Western and Chinese social media. Demonstrates bilingual (en/zh) capability. Both have unique content moderation challenges (duets, zhongcao, daigou).\n\n## DEC-003: 5-Dimension Rubric\n**Date:** 2026-04-22\n**Status:** Accepted\n**Decision:** accuracy (0.25), calibration (0.20), cultural_sensitivity (0.20), handoff_quality (0.20), explanation_quality (0.15).\n**Rationale:** Balanced weighting that emphasizes accuracy while giving significant weight to often-overlooked dimensions (cultural sensitivity, handoff quality).\n\n## DEC-004: LLM-as-Judge Primary Evaluator\n**Date:** 2026-04-22\n**Status:** Accepted\n**Decision:** Use Claude / GPT-4o as primary LLM-as-judge evaluators alongside traditional API providers.\n**Rationale:** LLM judges provide richer explanations and can evaluate nuanced edge cases that keyword/classifier systems miss.\n\n## DEC-005: Positioning Statement\n**Date:** 2026-04-22\n**Status:** Accepted\n**Decision:** \"I evaluate AI moderation systems for edge-case accuracy and human-AI handoff quality.\"\n**Rationale:** Defines unique value prop at intersection of T&S + AI eval. Not a standard T&S analyst or ML engineer role.\n\n## DEC-006: Notion + GitHub MCP Bridge\n**Date:** 2026-04-22\n**Status:** Accepted\n**Decision:** Use Nexus 6.0 agent to write code in Notion and push directly to GitHub via MCP.\n**Rationale:** Eliminates workflow fragmentation between spec (Notion) and code (GitHub).
